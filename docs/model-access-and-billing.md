# VGO AI 模型接入与计费说明

## 1. 目标

VGO AI 采用统一模型网关：

- 用户只需要一个 API Key
- 同一个 Base URL 调用全部站内模型
- 所有消耗统一记入账户余额、请求日志、后台统计

## 2. 用户调用方式

Base URL:

```text
https://vgoai.cn/api/v1/gateway/v1
```

鉴权头:

```text
Authorization: Bearer sk-your-api-key
```

常用接口:

- `GET /models`
- `POST /chat/completions`

## 3. 单 API Key 逻辑

用户在账户中心创建 API Key 后：

1. 同一个 Key 可以调用全部已开放模型
2. 用户不需要按模型分别申请 Key
3. 网关根据请求中的 `model` 字段自动选择可用渠道

## 4. 平台模型部署逻辑

后台新增模型时，按这个顺序配置：

1. 创建渠道
   - 渠道名称
   - 上游类型
   - Base URL
   - 上游 API Key
   - 优先级
   - 支持模型列表

2. 配置模型价格
   - 输入价格
   - 输出价格
   - 建议统一按每 100 万 tokens 录入

3. 测试渠道
   - 验证连通性
   - 检查上游返回
   - 观察日志状态

4. 正式启用
   - 开启渠道
   - 开启模型
   - 对外在模型目录中展示

## 5. 计费逻辑

当前系统按以下顺序计费：

1. 用户请求进入统一网关
2. 网关校验 API Key 是否有效
3. 网关读取账户余额
4. 根据 `model` 选择可用渠道
5. 调用上游模型
6. 根据 usage 统计：
   - `prompt_tokens`
   - `completion_tokens`
7. 按模型价格计算成本：
   - 输入 tokens 成本
   - 输出 tokens 成本
8. 乘以渠道倍率 `priceRate`
9. 扣减用户余额
10. 写入请求日志和统计数据

## 6. 成本公式

```text
输入成本 = (prompt_tokens / 1,000,000) * 输入价格
输出成本 = (completion_tokens / 1,000,000) * 输出价格
总成本 = (输入成本 + 输出成本) * 渠道倍率
```

## 7. 后台与用户端对应关系

用户端可以看到：

- API Key
- 可用模型
- 余额
- 请求日志
- token 消耗
- 充值与订单

后台可以看到：

- 渠道状态
- 模型路由
- 请求成本
- 用户调用记录
- 按模型统计
- 按渠道统计

## 8. 后续建议

建议下一步继续补齐：

- 模型单独上下架状态
- 模型展示名与真实上游模型 ID 分离
- 每模型单独售价与进价
- 渠道权重和故障切换
- 更完整的开发者 API 文档页
