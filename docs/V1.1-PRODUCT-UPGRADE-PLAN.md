# VGO CODE v1.1 产品化升级计划

## 目标
- 在 `v1.0.0` 可用基线上，完成可持续迭代的产品化升级。
- 完成去旧品牌化，统一为 `VGO CODE / VGO AI` 品牌体系。
- 提升功能稳定性、可观测性、可发布性，建立标准发布闸门。

## 基线结论
- 现有 `dist` 中 `VGO CODE Setup 1.0.0.exe` 可运行。
- `app.asar` 运行时代码与当前 `electron/server/ui` 源码一致。
- 已补齐 `scripts/afterPack.js`，恢复打包链路。

## 里程碑
### M1: 品牌治理（1 周）
- 执行 `npm run audit:brand` 生成品牌扫描报告。
- 清理 UI 文案、窗口标题、错误提示中的旧品牌词。
- 统一安装包名、更新源名称、产品标识文案。

### M2: 产品化能力（1-2 周）
- 首次启动引导：模型选择、账号绑定、网络检测。
- 健康检查中心：本地服务状态、模型可用性、CLI 可用性。
- 错误处理升级：将异常转为可理解的可操作提示。

### M3: 功能完善（1-2 周）
- 会话治理：多会话恢复、重命名、删除保护。
- 引擎治理：超时控制、重试策略、降级策略。
- 配置治理：环境检测、配置校验、配置迁移。

### M4: 测试与发布（1 周）
- 增加打包烟测、核心 API 烟测、UI 关键路径回归。
- 执行 `npm run verify:release` 作为发布前检查入口。
- 建立发布清单（Go/No-Go），避免“能打包但不可发布”。

## 自动化清单
- `npm run plan:next`：自动生成 `docs/NEXT-VERSION-PLAN.md`。
- `npm run audit:brand`：自动生成 `docs/BRAND-AUDIT-REPORT.md`。
- `npm run verify:release`：串行执行审计、规划、打包、产物校验。

## v1.1 版本验收标准
- 品牌扫描报告中的高风险旧品牌命中数为 `0`。
- 打包产物可生成，且桌面端可成功启动。
- 核心流程可用：新建会话、发送消息、模型切换、设置保存。
- 关键异常可提示下一步动作，不出现无信息崩溃。

## 版本节奏建议
- `v1.0.1`（修复版）：仅修复打包链路与高优缺陷。
- `v1.1.0`（产品化版）：品牌治理 + 体验升级 + 测试闸门。
- `v1.2.0`（能力版）：更完整引擎适配与生态扩展。
