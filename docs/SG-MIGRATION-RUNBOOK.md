# 新加坡定点 IP 迁移 Runbook（主域名不变）

适用场景：
- 域名注册商：腾讯云
- DNS 托管：Cloudflare
- 目标：业务迁移到新加坡机房，主域名保持不变

## 目标原则

1. 主域名不改，只改解析指向
2. 先预部署验证，再灰度切换
3. 任何时刻可回滚到旧源站

## 变更窗口前（T-24h）

1. 在 Cloudflare 将主域名相关记录 TTL 调低到 `120s`（或 `300s`）。
2. 记录当前线上 DNS 旧值（A/CNAME）到回滚清单。
3. 确认旧源站保留 24~48 小时可用。

## 新加坡环境准备（不切流）

1. 创建新加坡入口：
- 地域：Singapore（如 ap-southeast-1）
- 出口：固定公网 IP（EIP）或固定 LB 地址

2. 部署应用并完成健康检查：
- 健康接口：`/health`
- 核心流程：登录、会话、模型调用、上传下载（如有）

3. 绑定临时验证域名（Cloudflare 新增记录）：
- 示例：`sg-origin.yourdomain.com` -> 新加坡入口（A/CNAME）
- 证书校验通过后再进入灰度阶段

## Cloudflare 侧切换策略

## 方案 A（推荐，稳定）

使用 Cloudflare 代理（橙云）做主入口，主域名保持不变：

1. 主域名记录保持在 Cloudflare（橙云开启）。
2. 更新主域名回源目标到新加坡入口。
3. 如需灰度，使用 Cloudflare Rules 按路径/国家/比例分流。

优点：
- 可以快速回滚
- 对用户无感
- 可利用 Cloudflare 缓存/WAF/限流

## 方案 B（直连）

主域名 A 记录直接改为新加坡固定 IP（可开或关 Cloudflare 代理）。

注意：
- 如关闭代理，源站 IP 会暴露
- 回滚依赖 DNS 缓存刷新速度

## 切换执行（T 时刻）

1. 先切小流量（建议 5% 或指定路径）。
2. 观察 15~30 分钟：
- 5xx 错误率
- 接口超时
- 登录成功率
- 页面核心性能指标
3. 指标正常后扩大到 100%。

## 回滚预案（必须可一键执行）

触发条件（任一满足即回滚）：
- 错误率显著升高
- 登录/支付/回调等关键链路异常
- 超时持续超过阈值

回滚动作：
1. Cloudflare 记录回切到旧源站（或关闭灰度规则）。
2. 验证主域名恢复正常。
3. 记录故障时间点与错误样本，待修复后再切换。

## 验证清单

1. DNS 解析：
- 国内/东南亚/欧美分别验证解析与可达性

2. 证书：
- 证书链完整，SNI 正常，无过期告警

3. 业务：
- 登录、聊天、会话管理、设置保存、文件相关（如有）

4. 观测：
- 访问日志、应用日志、网关日志
- 错误告警、延迟告警
